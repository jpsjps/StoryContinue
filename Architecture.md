## 一、需求摘要

- **核心**：输入已有小说内容，由 AI 续写后续情节
- **结局风格**：可选（圆满、悲剧、开放式等）
- **部署**：本地部署，Linux 环境
- **交互**：网页界面
- **AI 接入**：兼容 OpenAI 协议 或 Anthropic 协议的 API（可切换/多后端）

### 3.2 技术选型建议（供讨论）

| 层次                 | 选项                                     | 说明                                               |
| -------------------- | ---------------------------------------- | -------------------------------------------------- |
| **后端**       | **Python + FastAPI**               | 异步、原生 SSE、OpenAPI，便于流式与多后端切换      |
| **前端**       | 单页 HTML + JS（可后续升级 Vue/React）   | 先单页实现项目列表、章节、流式展示区、设置         |
| **流式**       | **SSE (Server-Sent Events)**       | 单向推送即可，实现简单；WebSocket 可选后续         |
| **API 客户端** | `openai` SDK、`anthropic` SDK        | 适配层统一接口，均需使用各家的 stream 能力         |
| **配置**       | 环境变量 + 可选 `config.json` / 设置页 | API Key 环境变量或设置页持久化，不提交仓库         |
| **部署**       | 默认 `127.0.0.1`；可选 `0.0.0.0`     | 本机访问；需局域网时启动参数或配置绑定 `0.0.0.0` |

### 3.3 核心流程

**续写一章：**

1. 用户选择或创建「项目/书籍」，确认当前原文 + 已续写章节内容。
2. 选择「结局风格」→ 后端拼「系统 prompt + 风格描述 + 原文与已有续写」。
3. 请求「续写一章」→ 上下文处理（超长则截断/摘要）→ LLM 适配层 **stream 调用**。
4. 通过 **SSE** 流式推送到前端，前端逐字/逐段追加显示（打字机效果）。
5. 流结束后将本章写入当前项目，持久化。

**续写到结局：**

- 与上类似，但系统 prompt 中明确「本章为结局章，请在本章内收束故事并体现所选结局风格」；可限制最大 token 或提示模型控制篇幅。

### 3.4 项目/书籍数据模型（初版）

- **项目 (Book/Project)**：id、标题、创建/更新时间、关联章节列表。
- **内容**：每个项目包含「原文」（用户输入） + **已续写章节** 有序列表（每章一段文本）。
- **持久化**：本地 JSON 文件 或 SQLite。目录如 `data/projects/{project_id}.json` 或单库 `data/story.db`。

便于：多轮续写 = 在「原文 + 章1…章N」基础上续写章 N+1；「续写到结局」= 一次生成结局章。

### 3.5 流式技术选型：SSE

- **SSE (Server-Sent Events)**：HTTP 长连接，服务端单向推送。FastAPI 原生支持 `StreamingResponse` + `text/event-stream`。
- 前端 `EventSource` 或 `fetch` 读 stream，收到一块就追加到 DOM。
- 相比 WebSocket：实现简单、无需双向协议，满足「边生成边显示」即可。

### 3.6 结局风格（扩展列表）

| 风格   | Prompt 要点                            |
| ------ | -------------------------------------- |
| 圆满   | 结局积极、团圆、矛盾化解、善有善报     |
| 悲剧   | 遗憾、牺牲、不可逆的失去、命运感       |
| 开放式 | 留白、多义、不给出唯一解释、读者可遐想 |
| 惊悚   | 紧张、悬念、轻微恐怖感、可留余悸       |
| 悬疑   | 谜题收束、反转或揭示、逻辑闭环         |
| 治愈   | 温暖、和解、成长、情感抚慰             |

前端：下拉框 + 可选「自定义关键词」输入框；后端映射为一段「风格描述」拼进系统 prompt。
